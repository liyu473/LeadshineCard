# 插补控制器使用指南（小白版）

## 一、基础概念

### 1. 什么是插补？

**插补（Interpolation）** 就是让多个轴（电机）协调运动，形成特定的轨迹。

**举例说明：**
- **单轴运动**：就像一个电机单独动，比如X轴从0移动到100mm
- **多轴联动（插补）**：多个电机同时协调运动，比如X轴和Y轴同时动，画出一条直线或圆弧

### 2. 什么是缓冲区？

**缓冲区（Buffer）** 就像一个"任务队列"，你可以提前把很多运动指令放进去，板卡会按顺序执行。

**为什么需要缓冲区？**
- **单段插补**：发一条指令，等执行完，再发下一条 → 会有停顿
- **连续插补**：提前把多条指令放进缓冲区 → 运动连贯流畅，没有停顿

**类比：**
- 单段插补 = 你一句一句地告诉司机往哪开
- 连续插补 = 你提前把整个路线图给司机，他可以连续开

## 二、插补控制器的两种使用模式

### 模式1：单段插补（简单场景）

**适用场景：**
- 简单的点到点运动
- 不需要连续运动
- 轨迹简单

**使用步骤：**

```csharp
// 1. 创建插补控制器
var interpolation = new LeadshineInterpolationController(cardNo: 0);

// 2. 设置速度参数（必须先设置！）
await interpolation.SetVectorProfileAsync(
    crd: 0,           // 坐标系号
    minVel: 1.0,      // 最小速度 1 mm/s
    maxVel: 100.0,    // 最大速度 100 mm/s
    tacc: 0.2,        // 加速时间 0.2秒
    tdec: 0.2,        // 减速时间 0.2秒
    stopVel: 0.5      // 停止速度 0.5 mm/s
);

// 3. 执行直线插补（X轴和Y轴同时运动到目标位置）
ushort[] axes = new ushort[] { 0, 1 };  // 轴0(X轴)和轴1(Y轴)
double[] targetPos = new double[] { 100.0, 50.0 };  // X轴到100mm, Y轴到50mm

await interpolation.LineInterpolationAsync(axes, targetPos);

// 4. 等待运动完成
// （通常需要配合轴控制器检查运动状态）
```

### 模式2：连续插补（复杂轨迹）

**适用场景：**
- 需要画复杂图形（多条直线+圆弧组合）
- 要求运动连贯流畅
- 加工、绘图等应用

**使用步骤：**

```csharp
// 1. 创建插补控制器
var interpolation = new LeadshineInterpolationController(cardNo: 0);

// 2. 设置速度参数
await interpolation.SetVectorProfileAsync(
    crd: 0, 
    minVel: 1.0, 
    maxVel: 100.0, 
    tacc: 0.2, 
    tdec: 0.2, 
    stopVel: 0.5
);

// 3. 打开连续插补缓冲区（重要：指定参与插补的轴）
ushort[] axes = new ushort[] { 0, 1 };  // 轴0(X轴)和轴1(Y轴)
await interpolation.OpenContinuousBufferAsync(crd: 0, axes);

// 注意：OpenContinuousBufferAsync 会缓存轴配置
// 后续 AddLineSegmentAsync/AddArcSegmentAsync 会自动使用这个轴配置
// targetPositions 数组的长度必须与 axes 数组长度一致

// 4. 添加多个运动段到缓冲区
// 第1段：直线到(100, 0)
// targetPositions[0]=100 对应 axes[0]=X轴
// targetPositions[1]=0 对应 axes[1]=Y轴
await interpolation.AddLineSegmentAsync(
    crd: 0,
    targetPositions: new double[] { 100.0, 0.0 },
    mark: 1  // 段标号，用于追踪进度
);

// 第2段：直线到(100, 100)
await interpolation.AddLineSegmentAsync(
    crd: 0, 
    targetPositions: new double[] { 100.0, 100.0 }, 
    mark: 2
);

// 第3段：圆弧到(0, 100)，圆心在(50, 100)
await interpolation.AddArcSegmentAsync(
    crd: 0,
    targetPositions: new double[] { 0.0, 100.0 },
    centerPositions: new double[] { 50.0, 100.0 },
    clockwise: true,  // 顺时针
    mark: 3
);

// 第4段：直线回到原点(0, 0)
await interpolation.AddLineSegmentAsync(
    crd: 0, 
    targetPositions: new double[] { 0.0, 0.0 }, 
    mark: 4
);

// 5. 关闭缓冲区（表示不再添加新段）
await interpolation.CloseContinuousBufferAsync(crd: 0);
// 注意：关闭缓冲区时会自动清除轴配置缓存

// 6. 开始执行连续插补
await interpolation.StartContinuousAsync(crd: 0);

// 7. 监控执行进度（可选）
while (true)
{
    // 检查是否完成
    bool isDone = await interpolation.CheckContinuousDoneAsync(crd: 0);
    if (isDone) break;
    
    // 查看当前执行到哪一段
    int currentMark = await interpolation.GetCurrentMarkAsync(crd: 0);
    Console.WriteLine($"正在执行第 {currentMark} 段");
    
    await Task.Delay(100);  // 等待100ms再检查
}

Console.WriteLine("连续插补完成！");
```

## 三、各个函数的使用场景

### 1. 速度参数设置（必须先调用）

```csharp
// 设置速度参数
await interpolation.SetVectorProfileAsync(crd, minVel, maxVel, tacc, tdec, stopVel);

// 获取当前速度参数
var profile = await interpolation.GetVectorProfileAsync(crd);
if (profile.HasValue)
{
    Console.WriteLine($"最大速度: {profile.Value.maxVel}");
}
```

**使用场景：**
- 在执行任何插补之前必须先设置
- 控制运动的速度和加减速特性
- 不同的工艺可能需要不同的速度参数

### 2. 直线插补

```csharp
// 单段直线插补
await interpolation.LineInterpolationAsync(axes, targetPositions);

// 连续插补中添加直线段
await interpolation.AddLineSegmentAsync(crd, targetPositions, mark);
```

**使用场景：**
- 两点之间的直线运动
- 矩形、多边形的边
- 简单的点到点移动

### 3. 圆弧插补（三种方式）

#### 方式1：圆心终点式（最常用）

```csharp
// 单段圆弧
await interpolation.ArcInterpolationAsync(
    axes, 
    targetPositions,    // 终点位置
    centerPositions,    // 圆心位置（相对于起点）
    clockwise: true     // 顺时针/逆时针
);

// 连续插补中添加圆弧段
await interpolation.AddArcSegmentAsync(crd, targetPositions, centerPositions, clockwise, mark);
```

**使用场景：**
- 知道圆心位置时使用
- 最精确的圆弧定义方式

#### 方式2：半径终点式

```csharp
// 单段圆弧
await interpolation.ArcInterpolationByRadiusAsync(
    axes,
    targetPositions,    // 终点位置
    arcRadius: 50.0,    // 圆弧半径
    clockwise: true,
    circle: 0           // 0=圆弧, >0=整圆圈数
);

// 连续插补中添加圆弧段
await interpolation.AddArcSegmentByRadiusAsync(crd, targetPositions, arcRadius, clockwise, circle, mark);
```

**使用场景：**
- 知道半径但不知道圆心时使用
- 注意：起点到终点的距离必须 ≤ 2×半径

#### 方式3：三点式

```csharp
// 单段圆弧
await interpolation.ArcInterpolationBy3PointsAsync(
    axes,
    targetPositions,    // 终点位置
    midPositions,       // 中间点位置
    circle: 0
);

// 连续插补中添加圆弧段
await interpolation.AddArcSegmentBy3PointsAsync(crd, targetPositions, midPositions, circle, mark);
```

**使用场景：**
- 通过三个点定义圆弧
- 适合从测量数据生成圆弧

### 4. 圆弧限制设置

```csharp
// 设置圆弧限制
await interpolation.SetArcLimitAsync(
    crd: 0,
    enable: true,           // 启用限制
    maxCenAcc: 1000.0,      // 最大向心加速度 mm/s²
    maxArcError: 0.01       // 最大圆弧误差 mm
);
```

**使用场景：**
- 控制圆弧运动的质量
- 防止速度过快导致的轨迹偏差
- 高精度加工时必须设置

### 5. 弧长计算（辅助工具）

```csharp
// 计算圆心式圆弧的弧长
double? length = await interpolation.CalculateArcLengthByCenterAsync(
    startPos, targetPos, cenPos, clockwise, circle
);

// 计算三点式圆弧的弧长
double? length = await interpolation.CalculateArcLengthBy3PointsAsync(
    startPos, midPos, targetPos, circle
);

// 计算半径式圆弧的弧长
double? length = await interpolation.CalculateArcLengthByRadiusAsync(
    startPos, targetPos, arcRadius, clockwise, circle
);
```

**使用场景：**
- 估算运动时间
- 计算加工长度
- 优化运动路径

### 6. 连续插补控制

```csharp
// 暂停连续插补
await interpolation.PauseContinuousAsync(crd);

// 继续执行（暂停后恢复）
await interpolation.StartContinuousAsync(crd);

// 停止连续插补
await interpolation.StopContinuousAsync(crd);

// 查询剩余缓冲区空间
int space = await interpolation.GetRemainingBufferSpaceAsync(crd);
Console.WriteLine($"缓冲区还能放 {space} 个运动段");
```

**使用场景：**
- 暂停：用户按下暂停按钮
- 停止：紧急停止或任务取消
- 查询空间：动态添加运动段时，确保缓冲区不满

## 四、完整示例：画一个矩形

```csharp
public async Task DrawRectangleAsync()
{
    var interpolation = new LeadshineInterpolationController(cardNo: 0);
    
    // 1. 设置速度参数
    await interpolation.SetVectorProfileAsync(0, 1.0, 50.0, 0.2, 0.2, 0.5);
    
    // 2. 打开连续插补缓冲区
    ushort[] axes = new ushort[] { 0, 1 };  // X轴和Y轴
    await interpolation.OpenContinuousBufferAsync(0, axes);
    
    // 3. 添加矩形的四条边
    await interpolation.AddLineSegmentAsync(0, new double[] { 100, 0 }, 1);    // 右
    await interpolation.AddLineSegmentAsync(0, new double[] { 100, 100 }, 2);  // 上
    await interpolation.AddLineSegmentAsync(0, new double[] { 0, 100 }, 3);    // 左
    await interpolation.AddLineSegmentAsync(0, new double[] { 0, 0 }, 4);      // 下
    
    // 4. 关闭缓冲区并开始执行
    await interpolation.CloseContinuousBufferAsync(0);
    await interpolation.StartContinuousAsync(0);
    
    // 5. 等待完成
    while (!await interpolation.CheckContinuousDoneAsync(0))
    {
        await Task.Delay(100);
    }
    
    Console.WriteLine("矩形绘制完成！");
}
```

## 五、常见问题

### Q1: 什么时候用单段插补，什么时候用连续插补？

**单段插补：**
- 简单的点到点运动
- 不在乎运动之间的停顿
- 轨迹简单

**连续插补：**
- 复杂图形（多条直线+圆弧）
- 要求运动流畅连贯
- 加工、绘图等应用

### Q2: 坐标系号（crd）是什么？

坐标系号用于区分不同的插补任务。一个板卡可以有多个坐标系（通常0-3），每个坐标系可以独立执行插补运动。

**举例：**
- 坐标系0：控制XY平台
- 坐标系1：控制旋转轴

### Q3: 段标号（mark）有什么用？

段标号用于追踪执行进度。通过 [`GetCurrentMarkAsync`](LeadshineCard/Core/Interfaces/IInterpolationController.cs:96) 可以知道当前执行到哪一段，方便监控和调试。

### Q4: 为什么要先设置速度参数？

速度参数决定了运动的快慢和加减速特性。如果不设置，板卡会使用默认值，可能导致运动过快或过慢。

### Q5: 如果忘记打开缓冲区就添加段会怎样？

**v2.0 改进**：会立即抛出 `InvalidOperationException` 异常，提示"坐标系 X 未打开，请先调用 OpenContinuousBufferAsync"。

```csharp
// ❌ 错误示例
try
{
    await interpolation.AddLineSegmentAsync(crd: 0, targetPositions: new double[] { 100, 200 });
}
catch (InvalidOperationException ex)
{
    Console.WriteLine(ex.Message);
    // 输出: 坐标系 0 未打开，请先调用 OpenContinuousBufferAsync
}
```

### Q6: targetPositions 数组的顺序是什么？

`targetPositions` 数组的顺序与 `OpenContinuousBufferAsync` 中 `axes` 数组的顺序一致：

```csharp
// 打开缓冲区时指定轴顺序
await interpolation.OpenContinuousBufferAsync(
    crd: 0,
    axes: new ushort[] { 2, 0, 1 }  // 注意：轴2, 轴0, 轴1
);

// 添加段时，targetPositions 的顺序对应 axes 的顺序
await interpolation.AddLineSegmentAsync(
    crd: 0,
    targetPositions: new double[] { 100, 200, 300 }
    // targetPositions[0]=100 → axes[0]=轴2
    // targetPositions[1]=200 → axes[1]=轴0
    // targetPositions[2]=300 → axes[2]=轴1
);
```

### Q7: 缓冲区满了怎么办？

**v2.0 改进**：`Add*Segment` 方法会自动检查缓冲区空间，如果空间不足会自动等待（最多5秒）。如果5秒后仍然没有空间，会抛出 `MotionCardException` 异常。

```csharp
try
{
    // 自动等待缓冲区空间
    await interpolation.AddLineSegmentAsync(crd: 0, targetPositions: new double[] { 100, 200 });
}
catch (MotionCardException ex)
{
    Console.WriteLine($"缓冲区超时: {ex.Message}");
}
```

## 六、注意事项

### 1. 轴配置缓存机制（重要！）

**v2.0 新增优化**：插补控制器会自动缓存坐标系的轴配置，简化使用流程。

#### 工作原理：

```csharp
// 1. 打开缓冲区时指定轴配置
ushort[] axes = new ushort[] { 0, 1, 2 };  // X, Y, Z 三轴
await interpolation.OpenContinuousBufferAsync(crd: 0, axes);
// ✅ 轴配置被缓存：坐标系0 → [0, 1, 2]

// 2. 添加段时只需提供目标位置，轴配置自动使用缓存
await interpolation.AddLineSegmentAsync(
    crd: 0,
    targetPositions: new double[] { 100, 200, 300 }  // 对应 X=100, Y=200, Z=300
);
// ✅ 自动从缓存获取轴配置 [0, 1, 2]

// 3. 关闭缓冲区时自动清除缓存
await interpolation.CloseContinuousBufferAsync(crd: 0);
// ✅ 清除坐标系0的轴配置缓存
```

#### 重要规则：

1. **必须先打开缓冲区**：
   ```csharp
   // ❌ 错误：未打开缓冲区就添加段
   await interpolation.AddLineSegmentAsync(crd: 0, targetPositions: new double[] { 100, 200 });
   // 抛出异常：InvalidOperationException: 坐标系 0 未打开

   // ✅ 正确：先打开缓冲区
   await interpolation.OpenContinuousBufferAsync(crd: 0, new ushort[] { 0, 1 });
   await interpolation.AddLineSegmentAsync(crd: 0, targetPositions: new double[] { 100, 200 });
   ```

2. **目标位置数组长度必须匹配**：
   ```csharp
   // 打开缓冲区时指定了2个轴
   await interpolation.OpenContinuousBufferAsync(crd: 0, new ushort[] { 0, 1 });

   // ❌ 错误：目标位置只有1个
   await interpolation.AddLineSegmentAsync(crd: 0, targetPositions: new double[] { 100 });
   // 抛出异常：目标位置数组长度(1)必须与坐标系轴数(2)相同

   // ✅ 正确：目标位置有2个
   await interpolation.AddLineSegmentAsync(crd: 0, targetPositions: new double[] { 100, 200 });
   ```

3. **支持非连续轴号**：
   ```csharp
   // 可以使用任意轴号组合
   await interpolation.OpenContinuousBufferAsync(crd: 0, new ushort[] { 0, 2, 5 });
   // 坐标系0使用轴0、轴2、轴5

   await interpolation.AddLineSegmentAsync(
       crd: 0,
       targetPositions: new double[] { 100, 200, 300 }
       // targetPositions[0]=100 → 轴0
       // targetPositions[1]=200 → 轴2
       // targetPositions[2]=300 → 轴5
   );
   ```

4. **多坐标系独立缓存**：
   ```csharp
   // 坐标系0：XY平面
   await interpolation.OpenContinuousBufferAsync(crd: 0, new ushort[] { 0, 1 });

   // 坐标系1：YZ平面
   await interpolation.OpenContinuousBufferAsync(crd: 1, new ushort[] { 1, 2 });

   // 两个坐标系的轴配置互不影响
   await interpolation.AddLineSegmentAsync(crd: 0, targetPositions: new double[] { 100, 200 });
   await interpolation.AddLineSegmentAsync(crd: 1, targetPositions: new double[] { 300, 400 });
   ```

### 2. 缓冲区自动管理

**v2.0 新增优化**：添加段时自动检查缓冲区空间，无需手动管理。

```csharp
// 旧版本（v1.x）需要手动检查
int space = await interpolation.GetRemainingBufferSpaceAsync(crd: 0);
if (space < 1)
{
    await Task.Delay(100);  // 等待空间
}
await interpolation.AddLineSegmentAsync(...);

// 新版本（v2.0）自动检查和等待
await interpolation.AddLineSegmentAsync(...);
// ✅ 内部自动检查缓冲区，空间不足时自动等待（最多5秒）
```

### 3. 统一的异常处理

**v2.0 改进**：所有 `Add*Segment` 方法统一抛出异常，而不是返回 false。

```csharp
try
{
    await interpolation.AddLineSegmentAsync(crd: 0, targetPositions: new double[] { 100, 200 });
    await interpolation.AddArcSegmentAsync(crd: 0, ...);
}
catch (InvalidOperationException ex)
{
    // 坐标系未打开
    Console.WriteLine($"错误: {ex.Message}");
}
catch (ArgumentException ex)
{
    // 参数错误（如数组长度不匹配）
    Console.WriteLine($"参数错误: {ex.Message}");
}
catch (MotionCardException ex)
{
    // 板卡操作失败
    Console.WriteLine($"板卡错误: {ex.Message}, 错误码: {ex.ErrorCode}");
}
```

### 4. 其他注意事项

1. **必须先设置速度参数**，否则运动可能异常
2. **连续插补必须按顺序**：打开缓冲区 → 添加段 → 关闭缓冲区 → 开始执行
3. **坐标位置是绝对位置**，不是相对位置（除非使用相对模式）
4. **圆弧插补要注意方向**：顺时针/逆时针会影响轨迹
5. **缓冲区有容量限制**，虽然会自动等待，但不要一次添加过多段
