# 快速开始 - 优化版功能

## 5分钟上手优化功能

### 1. 使用预设参数（最简单）

```csharp
using LeadshineCard.Core.Models;
using LeadshineCard.Implementation;

var card = new LeadshineMotionCard();
await card.InitializeAsync(0);

var axis = card.GetAxisController(0);

// 一行代码设置参数
await axis.SetMotionParametersAsync(MotionParametersPresets.Balanced);

// 运动 + 等待完成
await axis.MoveAbsoluteAsync(100.0);
await axis.WaitMotionCompleteAsync();
```

### 2. 添加软限位保护（推荐）

```csharp
using LeadshineCard.Core.Models;

// 设置软限位，防止碰撞
var softLimit = new SoftLimit
{
    Enabled = true,
    PositiveLimit = 500.0,
    NegativeLimit = -10.0
};
axis.SetSoftLimit(softLimit);

// 现在运动会自动检查限位
await axis.MoveAbsoluteAsync(100.0);
await axis.WaitMotionCompleteAsync();
```

### 3. 使用事件监控（避免轮询）

```csharp
using LeadshineCard.Core.Events;

// 订阅运动完成事件
axis.MotionCompleted += (s, e) =>
{
    Console.WriteLine($"运动完成，位置: {e.FinalPosition:F2}mm");
};

// 订阅限位事件
axis.LimitTriggered += (s, e) =>
{
    Console.WriteLine($"触发{(e.IsPositiveLimit ? "正" : "负")}限位！");
};

// 执行运动，事件自动触发
await axis.MoveAbsoluteAsync(100.0);
await axis.WaitMotionCompleteAsync();
```

### 4. 支持取消的回零

```csharp
using System.Threading;

var cts = new CancellationTokenSource();

// 设置回零
await axis.SetHomePositionAsync(0.0);
await axis.SetHomeModeAsync(HomeDirection.Negative, 10.0, HomeMode.HomeSwitch);
await axis.HomeMoveAsync();

// 等待回零，支持取消
try
{
    bool success = await axis.WaitHomeCompleteAsync(30000, cts.Token);
    Console.WriteLine(success ? "回零成功" : "回零失败");
}
catch (OperationCanceledException)
{
    Console.WriteLine("回零被取消");
}
```

### 5. 完整示例

```csharp
using LeadshineCard.Core.Models;
using LeadshineCard.Core.Enums;
using LeadshineCard.Core.Events;
using LeadshineCard.Implementation;

// 初始化
var card = new LeadshineMotionCard();
await card.InitializeAsync(0);
var axis = card.GetAxisController(0);

// 1. 设置软限位
axis.SetSoftLimit(new SoftLimit
{
    Enabled = true,
    PositiveLimit = 500.0,
    NegativeLimit = -10.0
});

// 2. 使用预设参数
await axis.SetMotionParametersAsync(MotionParametersPresets.Balanced);

// 3. 订阅事件
axis.MotionCompleted += (s, e) => Console.WriteLine($"完成: {e.FinalPosition:F2}mm");
axis.HomeCompleted += (s, e) => Console.WriteLine($"回零{(e.Success ? "成功" : "失败")}");

// 4. 回零
await axis.SetHomePositionAsync(0.0);
await axis.SetHomeModeAsync(HomeDirection.Negative, 10.0, HomeMode.HomeSwitch);
await axis.HomeMoveAsync();
await axis.WaitHomeCompleteAsync();

// 5. 运动
await axis.MoveAbsoluteAsync(100.0);
await axis.WaitMotionCompleteAsync();

await axis.MoveAbsoluteAsync(200.0);
await axis.WaitMotionCompleteAsync();

// 6. 查询状态（并行查询，速度快3-5倍）
var status = await axis.GetStatusAsync();
Console.WriteLine($"位置: {status.Position:F2}mm, 速度: {status.Speed:F2}mm/s");

// 7. 关闭
await card.CloseAsync();
```

## 对比：优化前 vs 优化后

### 优化前的代码
```csharp
// 手动设置参数
var parameters = new MotionParameters
{
    PulseEquivalent = 0.001,
    MinSpeed = 0.5,
    MaxSpeed = 100.0,
    Acceleration = 500.0,
    Deceleration = 500.0,
    StopSpeed = 0.2,
    SCurveTime = 0.05
};
await axis.SetMotionParametersAsync(parameters);

// 手动轮询等待
await axis.MoveAbsoluteAsync(100.0);
while (!await axis.CheckDoneAsync())
{
    await Task.Delay(100);
}

// 手动检查 PVT 缓冲区
var space = await axis.GetPvtRemainSpaceAsync();
if (space < 10)
{
    await Task.Delay(100);
}
await axis.SetPvtTableAsync(times, positions, velocities);
```

### 优化后的代码
```csharp
// 使用预设
await axis.SetMotionParametersAsync(MotionParametersPresets.Balanced);

// 一行等待
await axis.MoveAbsoluteAsync(100.0);
await axis.WaitMotionCompleteAsync();

// 自动检查缓冲区
await axis.SetPvtTableAsync(times, positions, velocities);
```

## 常见问题

**Q: 需要修改现有代码吗？**
A: 不需要，完全向后兼容。新功能是可选的。

**Q: 软限位会影响性能吗？**
A: 几乎没有影响，只在运动前检查一次。

**Q: 事件会影响实时性吗？**
A: 不会，事件是异步触发的，不阻塞主流程。

**Q: 参数缓存会导致数据不一致吗？**
A: 缓存只有5秒，且只缓存自己设置的参数。如果担心，可以手动清除缓存（重新设置参数）。

**Q: 可以只使用部分优化功能吗？**
A: 可以，所有功能都是独立的，按需使用。
